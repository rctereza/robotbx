Automating User Interface Interactions (Desktop Automation):

    The java.awt.Robot class in the Java AWT package enables programmatic control of the mouse and keyboard.
    This is primarily used for test automation, creating self-running demonstrations, or other applications requiring simulated user input.
    It allows for actions like typing text, clicking buttons, and moving the mouse cursor, effectively automating interactions with desktop applications.


The Robot class in the Java AWT package is used to generate native system input events for the purposes of test automation, self-running demos, 
and other applications where control of the mouse and keyboard is needed. The primary purpose of Robot is to facilitate automated testing of Java platform implementations.

import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.io.IOException;

public class CommandExecutor {
    public static void main(String[] args) {
        try {
            // Command to execute (e.g., list files in current directory)
            String[] command;
            if (System.getProperty("os.name").toLowerCase().contains("win")) {
                command = new String[]{"cmd.exe", "/c", "dir"}; // Windows
            } else {
                command = new String[]{"ls", "-l"}; // Linux/macOS
            }

            Process process = Runtime.getRuntime().exec(command);

            // Read output from the command
            BufferedReader stdInput = new BufferedReader(new InputStreamReader(process.getInputStream()));
            String s;
            System.out.println("Standard Output:");
            while ((s = stdInput.readLine()) != null) {
                System.out.println(s);
            }

            // Read any errors from the command
            BufferedReader stdError = new BufferedReader(new InputStreamReader(process.getErrorStream()));
            System.out.println("Standard Error (if any):");
            while ((s = stdError.readLine()) != null) {
                System.out.println(s);
            }

            // Wait for the process to complete and get the exit code
            int exitCode = process.waitFor();
            System.out.println("Process exited with code: " + exitCode);

        } catch (IOException | InterruptedException e) {
            e.printStackTrace();
        }
    }
}

11. Using Java Accessibility API (for Java Applications)
If you need to interact with Java-based GUIs (like JFrame, JDialog, etc.), the Java Accessibility API can be helpful. The Accessibility API allows you to query the properties of Java applications, including their window dimensions.
However, this only works for Java-based applications. It will not work for native (non-Java) applications like Notepad or browsers.

Example: Using Java Accessibility API for Java Applications

import javax.accessibility.*;
import javax.swing.*;
import java.awt.*;

public class WindowDimensions {
    public static void main(String[] args) {
        try {
            // Create a simple JFrame for demo
            JFrame frame = new JFrame("Test Window");
            frame.setSize(800, 600);
            frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
            frame.setVisible(true);
            
            // Give some time for the JFrame to initialize
            Thread.sleep(1000);

            // Retrieve the window's Accessibility information
            AccessibleContext ac = frame.getAccessibleContext();
            Accessible accessible = ac.getAccessibleChild(0); // the child of the main frame
            
            // Get the position and size
            Rectangle bounds = frame.getBounds();
            System.out.println("Window Position: " + bounds.x + "," + bounds.y);
            System.out.println("Window Size: " + bounds.width + "x" + bounds.height);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Limitations:
This only works for Java applications.
It does not work for non-Java applications such as Notepad, Chrome, etc.

12. Using JNA (Java Native Access) for Native Applications
To interact with non-Java applications (like Windows apps), you can use JNA (Java Native Access), which provides a way to call native system libraries and APIs from Java.
You can use JNA to call Windows APIs like FindWindow and GetWindowRect to retrieve the dimensions of application windows.
Steps:
Add the JNA library to your project.
Use the Windows API functions to get window handles and their dimensions.
Example: Using JNA to Get Window Dimensions on Windows
First, add the JNA library to your project (you can get the JNA library from JNA's GitHub repository https://github.com/java-native-access/jna)

import com.sun.jna.*;
import com.sun.jna.win32.*;

public class GetWindowDimensions {
    // Define User32 interface to interact with Windows API
    public interface User32 extends StdCallLibrary {
        User32 INSTANCE = (User32) Native.load("user32", User32.class);

        int FindWindowA(String lpClassName, String lpWindowName);
        boolean GetWindowRect(int hwnd, RECT rect);
    }

    // Define RECT structure for window position and size
    public static class RECT extends Structure {
        public int left;
        public int top;
        public int right;
        public int bottom;

        @Override
        protected List getFieldOrder() {
            return Arrays.asList("left", "top", "right", "bottom");
        }
    }

    public static void main(String[] args) {
        // Find a window by its title
        String windowTitle = "Untitled - Notepad"; // Replace with the title of the target window
        int hwnd = User32.INSTANCE.FindWindowA(null, windowTitle);

        if (hwnd != 0) {
            RECT rect = new RECT();
            if (User32.INSTANCE.GetWindowRect(hwnd, rect)) {
                // Calculate width and height
                int width = rect.right - rect.left;
                int height = rect.bottom - rect.top;

                System.out.println("Window Dimensions:");
                System.out.println("Width: " + width);
                System.out.println("Height: " + height);
            } else {
                System.out.println("Failed to get window dimensions.");
            }
        } else {
            System.out.println("Window not found.");
        }
    }
}

Explanation:
JNA Interface: We define the User32 interface to access the Windows API functions (FindWindowA and GetWindowRect).
FindWindowA: This function finds a window by its title (windowTitle in this case).
GetWindowRect: This function retrieves the dimensions of the window (stored in a RECT structure).
RECT: This structure holds the window's position and dimensions (left, top, right, bottom).

Steps to Run the JNA Example:
Add JNA to your project (you can download the JNA jar from the JNA GitHub repository
Replace the windowTitle with the title of the window you want to interact with (e.g., "Untitled - Notepad" for Notepad).
Run the Java code, and it will print the window's width and height.

Limitations:
This is Windows-specific. For macOS or Linux, you would need to use platform-specific native libraries (e.g., using AppleScript for macOS or xwininfo for Linux).
Requires JNA: JNA allows you to access native libraries, so the code depends on having this library included.

Getting All Open Windows (Using JNA)
You can list all open windows and query their dimensions using EnumWindows and GetWindowRect.
Here’s an example to list all open windows and their dimensions:

import com.sun.jna.*;
import com.sun.jna.win32.*;

public class ListWindows {
    public interface User32 extends StdCallLibrary {
        User32 INSTANCE = (User32) Native.load("user32", User32.class);

        int EnumWindows(WndProc wndProc, Pointer arg);
        boolean GetWindowRect(int hwnd, RECT rect);

        interface WndProc extends Callback {
            boolean invoke(int hwnd, Pointer arg);
        }
    }

    public static class RECT extends Structure {
        public int left;
        public int top;
        public int right;
        public int bottom;

        @Override
        protected List getFieldOrder() {
            return Arrays.asList("left", "top", "right", "bottom");
        }
    }

    public static void main(String[] args) {
        User32.INSTANCE.EnumWindows(new User32.WndProc() {
            @Override
            public boolean invoke(int hwnd, Pointer arg) {
                RECT rect = new RECT();
                if (User32.INSTANCE.GetWindowRect(hwnd, rect)) {
                    // Output window dimensions
                    int width = rect.right - rect.left;
                    int height = rect.bottom - rect.top;

                    System.out.println("Window Handle: " + hwnd);
                    System.out.println("Width: " + width + ", Height: " + height);
                }
                return true;  // Continue enumerating
            }
        }, null);
    }
}

This code uses EnumWindows to enumerate all open windows and prints the dimensions of each window.

Conclusion:
For Java Applications: You can use the Java Accessibility API to query window dimensions if the window is a Java-based application.
For Native Applications (Windows): You can use JNA to access Windows API functions (FindWindow, GetWindowRect, etc.) to get the dimensions of other application windows.
Platform-Specific: The JNA approach is Windows-specific, and you would need different tools for macOS or Linux.
